# AGENTS.md

Guidance for Claude Code when working in this repository.

## Purpose

`github.com/instill-ai/x` is a shared Go library used across Instill AI backend services (api-gateway, pipeline-backend, mgmt-backend, artifact-backend, agent-backend, etc.). It provides common utilities, clients, and infrastructure helpers.

Per the README: "Breaking change may happen frequently. Please use it with caution." Downstream repos pin a specific version in their `go.mod`.

## Module

- Module path: `github.com/instill-ai/x`
- Go version: `1.25.6` (see `go.mod`)
- No `Makefile`, no `main` package — pure library.

## Packages

Top-level packages, each independently importable:

- `acl/` — access-control list helpers (OpenFGA-based).
- `blobstorage/` — blob storage abstractions.
- `checkfield/` — field validation / immutability checks for API payloads.
- `client/` — gRPC client construction, TLS, connection pooling, metadata propagation.
- `constant/` — shared constants.
- `errors/` — error types and gRPC status mapping.
- `file/` — file helpers.
- `log/` — zap-based structured logging, OTEL log bridge.
- `minio/` — MinIO/S3 client wrapper.
- `otel/` — OpenTelemetry trace/metric/log setup (OTLP gRPC exporters).
- `paginate/` — pagination token helpers.
- `resource/` — resource naming / ID helpers.
- `server/` — gRPC/HTTP server middleware, marshalers, interceptors.
- `temporal/` — Temporal client/worker setup with tally metrics.
- `mock/` — generated mocks (see below).
- `release-please/` — release automation config.

## Verified commands

Run from the repo root:

- `go build ./...` — build all packages.
- `go test ./...` — run tests (uses `quicktest`, `testify`, `minimock`).
- `go vet ./...` — static checks.
- `go mod tidy` — sync dependencies.
- `go generate ./mock/...` — regenerate mocks. Requires `minimock` on `PATH`:
  `go install github.com/gojuno/minimock/v3/cmd/minimock@latest`.

## Conventions

- Mocks are generated by `minimock` and live under `mock/<pkg>/` with the suffix `_mock.gen.go`. The canonical list of generated interfaces is in `mock/generator.go` — add new `//go:generate` directives there rather than scattering them.
- Test framework: mixed `frankban/quicktest` and `stretchr/testify`; match the style of neighbouring `_test.go` files in the package you edit.
- Keep packages independent; avoid cross-package imports within `x/` unless unavoidable.
- Because this library is imported by many services, treat public API changes as breaking and surface them in `CHANGELOG.md` (release-please managed).
